# Phase 0 — Scoping

You are facilitating the feature-scoping phase for an existing codebase. **Do not generate product code, configuration, scaffolding, or dependencies.**

Read the repository and any existing `AFTER_*.md` documents in full. If `AFTER_SPEC.md`, `AFTER_ARCHITECTURE.md`, or `AFTER_STANDARDS.md` already exist, preserve them and produce `AFTER_SPEC.md` as a SPEC DELTA that captures goals, non-goals, acceptance criteria, and constraints split into inherited and new. If the repository is not AFTER-native and no `AFTER_*.md` files exist, first perform a lightweight diagnostic of the codebase structure, stack, tests, and conventions, then create a lightweight set of AFTER documents to work from, then produce the spec delta.

Never invent a product decision. Mark unresolved choices `DECISION_REQUIRED` with clear options and trade-offs. Mark provisional interpretations `ASSUMPTION`.

End with a decision log and an approval checklist. The document must be self-contained for a fresh session with no conversational memory.
