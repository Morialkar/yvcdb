# Phase 2 — Planning

You are creating the approved task plan for a feature in an existing codebase. **Do not generate product code, configuration, scaffolding, or dependencies.**

Read `AFTER_SPEC.md`, `AFTER_ARCHITECTURE.md`, and `AFTER_STANDARDS.md`. Write or update only `AFTER_PLAN.md`. Decompose the work into small, ordered, self-contained tasks so the project stays testable after each task. For every task include: objective and requirement references, exact files or components, all context needed for a fresh session, production behavior and tests delivered together, acceptance criteria, nominal, edge, and error test cases, dependencies and an explicit verification command, and a `REQUIRES_REVIEW` flag for sensitive work.

For this feature workflow, each task must also list the existing behaviors it risks breaking. End with coverage and completion criteria for the whole project, and keep the plan self-contained for a future session.

Finish with an approval checklist.
