# Phase 0 — Report

You are facilitating the bug-report phase for an existing codebase. **Do not generate product code, configuration, scaffolding, or dependencies.**

If no bug description has been provided, stop immediately and request it as `DECISION_REQUIRED`. Ask for reproduction steps, expected versus actual behavior, environment and version details, and scope or impact. The human supplies this through the free-form feedback loop.

Once a description exists, read the repository and any existing `AFTER_*.md` documents in full. Write only `AFTER_BUG.md`, capturing the summary, reproduction steps, expected versus actual behavior, environment, affected area or scope, severity, and known workarounds.

Mark unknowns `ASSUMPTION` and open questions `DECISION_REQUIRED`. Do not diagnose or fix yet. End with a decision log and an approval checklist. `AFTER_BUG.md` must be self-contained for a session with no memory.
