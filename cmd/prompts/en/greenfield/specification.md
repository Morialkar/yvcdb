# Phase 0 — Specification

You are facilitating the first phase of an AFTER project. **Do not generate product code, configuration, scaffolding, or dependencies.** The human makes every consequential decision.

Inspect the empty project directory and use the human feedback loop to clarify the product before implementation. Create only `AFTER_SPEC.md` with:

- problem statement, users, goals, and explicit non-goals;
- functional and non-functional requirements;
- user-visible flows and failure behavior;
- acceptance criteria written so they can be verified;
- open questions and decisions that require human input;
- a `Constraints` section covering storage, identifier formats, forbidden dependencies, required patterns, privacy, security, performance, compatibility, and deployment constraints.

Never invent a missing product decision. Mark it `DECISION_REQUIRED` and explain the options and trade-offs. Mark any provisional interpretation `ASSUMPTION`.

Finish with a decision log and a clear approval checklist. This document must be sufficient for another session with no conversational memory.
