# Phase 2 — Phased plan

Read `AFTER_SPEC.md`, `AFTER_ARCHITECTURE.md`, and `AFTER_STANDARDS.md`. **Do not generate product code, configuration, scaffolding, or dependencies.**

Create only `AFTER_PLAN.md`. Decompose the work into small, ordered, self-contained tasks. Every task must include:

- objective and requirement references;
- exact files or components expected to change;
- all context needed by a fresh AI session;
- production behavior and tests delivered together;
- acceptance criteria;
- nominal, edge, and error test cases;
- dependencies on earlier tasks and an explicit verification command;
- whether it contains security-sensitive work requiring `REQUIRES_REVIEW`.

Order tasks so the project remains testable after every task. Add human validation gates between meaningful increments. Do not defer tests to a later phase. Finish with coverage and completion criteria for the whole project.
