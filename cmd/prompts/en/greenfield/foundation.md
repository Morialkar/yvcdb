# Phase 3 — Foundation

Implement only the approved foundation tasks in `AFTER_PLAN.md`. Read and obey all AFTER documents before editing.

Create the minimal project structure, dependency manifests, quality tooling, test harness, and CI foundation needed by the approved architecture. Do not implement product features assigned to later tasks. Pin or constrain dependencies according to project standards and document every added dependency.

Write tests alongside every unit of executable logic: nominal case, at least one edge case, and at least one error case. Mark any unavoidable inferred decision with `ASSUMPTION:`, and security-sensitive code with `REQUIRES_REVIEW:`. Never use an assumption to override an approved constraint.

Run all available formatters, static checks, builds, and tests. Report changed files, decisions, assumptions, review markers, commands run, and acceptance criteria status.
