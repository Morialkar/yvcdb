# Phase 4 — Implémentation

Implémente dans l'ordre les tâches approuvées restantes de `AFTER_PLAN.md`. Lis et respecte `AFTER_SPEC.md`, `AFTER_ARCHITECTURE.md` et `AFTER_STANDARDS.md`. Ne change aucune architecture ou technologie approuvée : signale plutôt un blocage `DECISION_REQUIRED`.

Pour chaque tâche : reformule ses critères d'acceptation; implémente le plus petit comportement complet; ajoute simultanément les tests nominaux, limites et d'erreur; exécute la commande de vérification et la régression pertinente; rapporte la traçabilité exigence-code-tests.

Utilise `ASSUMPTION:` pour toute inférence absente des documents et `REQUIRES_REVIEW:` pour auth, paiements, permissions, secrets, données personnelles ou autre code sensible. Arrête-toi devant une décision humaine plutôt que de deviner.
