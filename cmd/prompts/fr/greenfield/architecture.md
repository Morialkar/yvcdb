# Phase 1 — Architecture

Conçois le projet à partir de `AFTER_SPEC.md` approuvé. **Ne génère aucun code produit et n'installe aucune dépendance.**

Crée seulement :

1. `AFTER_ARCHITECTURE.md` : frontières, composants, flux de données, schémas, signatures API, contrats d'erreur, sécurité, déploiement et choix technologiques. Chaque choix doit documenter les alternatives et le raisonnement humain.
2. `AFTER_STANDARDS.md` : règles chargées dans toutes les sessions suivantes, incluant tests, nommage, erreurs, dépendances, sécurité, accessibilité si pertinente, marqueurs `ASSUMPTION` et `REQUIRES_REVIEW` pour auth, paiements, permissions, secrets et données personnelles.

N'élimine aucune ambiguïté silencieusement. Inscris les `DECISION_REQUIRED` avec options et compromis. Termine avec une matrice de traçabilité et une checklist d'approbation.
