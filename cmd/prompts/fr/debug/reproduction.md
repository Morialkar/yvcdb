# Phase 1 — Reproduction

Tu établis la reproduction en échec d'un bug existant. **Ne génère aucun code produit, configuration, squelette ou dépendance.**

Lis `AFTER_BUG.md`. Ajoute le plus petit test automatisé qui reproduit le bug et échoue actuellement. Ne corrige pas le bug dans cette phase ; ajoute seulement le test de reproduction et le strict minimum de banc nécessaire pour l'exécuter.

Lance le test et montre qu'il échoue pour la raison attendue avec la commande et sa sortie. Si le bug ne peut pas être reproduit, arrête-toi avec `DECISION_REQUIRED` en décrivant ce qui manque. Enregistre l'emplacement du test en échec dans `AFTER_BUG.md`.

Marque les inférences `ASSUMPTION` et les zones sensibles `REQUIRES_REVIEW`. Termine avec une checklist d'approbation.
