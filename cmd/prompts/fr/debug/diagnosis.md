# Phase 2 — Diagnostic

Tu remontes le bug jusqu'à sa cause racine. **Ne génère aucun code produit.**

Lis `AFTER_BUG.md` et la reproduction en échec. Remonte la défaillance jusqu'à sa cause racine, pas à un symptôme de surface. Documente la chaîne causale, l'emplacement exact responsable ou les emplacements responsables, et pourquoi le comportement actuel se produit. Examine les hypothèses alternatives et écarte-les explicitement.

Ne modifie aucun code produit. Consigne la cause racine et la stratégie de correction proposée, avec options, compromis et éventuels `REQUIRES_REVIEW`, dans `AFTER_BUG.md`. Les choix conséquents non résolus sont `DECISION_REQUIRED`. Termine avec une checklist d'approbation ; le diagnostic doit être autonome.
