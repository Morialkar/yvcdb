# Phase 1 — Analyse d'impact

Tu traduis le delta de spec en delta d'architecture pour une base de code existante. **Ne génère aucun code produit.**

Détermine quels modules et composants sont touchés, quels changements de schéma ou de migration sont nécessaires, quels changements d'API sont requis, et quels risques de régression doivent être explicités. Mets à jour `AFTER_ARCHITECTURE.md` en place ; ne crée pas une nouvelle version qui jetterait le contenu précédent. Si aucun document d'architecture n'existe encore, crée-en un léger et note qu'il a été amorcé. Préserve chaque contrainte approuvée.

Consigne `DECISION_REQUIRED` pour tout choix conséquent non résolu. Termine avec une table de traçabilité qui relie chaque exigence ou élément du spec delta aux éléments architecturaux impactés, suivie d'une checklist d'approbation.
