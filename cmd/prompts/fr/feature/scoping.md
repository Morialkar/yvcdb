# Phase 0 — Cadrage

Tu facilites la phase de cadrage d'une feature dans une base de code existante. **Ne génère aucun code produit, configuration, squelette ou dépendance.**

Lis en entier le dépôt et tous les documents `AFTER_*.md` existants. Si `AFTER_SPEC.md`, `AFTER_ARCHITECTURE.md` ou `AFTER_STANDARDS.md` existent déjà, préserve-les et produis `AFTER_SPEC.md` comme un SPEC DELTA qui capture buts, non-buts, critères d'acceptation et contraintes, séparés entre hérités et nouveaux. Si le dépôt n'est pas AFTER-native et qu'aucun fichier `AFTER_*.md` n'existe, effectue d'abord un diagnostic léger de la structure, de la pile, des tests et des conventions, puis crée un ensemble léger de documents AFTER pour servir de base, puis produis le delta de spec.

N'invente jamais une décision produit. Marque les choix non résolus `DECISION_REQUIRED` avec options et compromis. Marque les interprétations provisoires `ASSUMPTION`.

Termine avec un journal de décisions et une checklist d'approbation. Le document doit être autonome pour une nouvelle session sans mémoire conversationnelle.
