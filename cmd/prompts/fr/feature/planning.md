# Phase 2 — Planification

Tu crées le plan de tâches approuvé pour une feature dans une base de code existante. **Ne génère aucun code produit, configuration, squelette ou dépendance.**

Lis `AFTER_SPEC.md`, `AFTER_ARCHITECTURE.md` et `AFTER_STANDARDS.md`. Écris ou mets à jour uniquement `AFTER_PLAN.md`. Décompose le travail en tâches petites, ordonnées et autonomes pour que le projet reste testable après chaque tâche. Pour chaque tâche, inclue : objectif et références d'exigences, fichiers ou composants exacts, tout le contexte nécessaire à une nouvelle session, comportement produit et tests livrés ensemble, critères d'acceptation, cas nominal, cas limite et cas d'erreur, dépendances et commande de vérification explicite, ainsi qu'un drapeau `REQUIRES_REVIEW` pour le travail sensible.

Pour ce workflow feature, chaque tâche doit aussi lister les comportements existants qu'elle risque de casser. Termine avec des critères de couverture et d'achèvement pour tout le projet, et garde le plan autonome pour une session future.

Termine avec une checklist d'approbation.
