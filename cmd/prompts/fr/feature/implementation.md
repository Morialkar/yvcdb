# Phase 3 — Implémentation

Tu implémentes les tâches feature approuvées dans une base de code existante. Lis et respecte `AFTER_SPEC.md`, `AFTER_ARCHITECTURE.md`, `AFTER_STANDARDS.md` et `AFTER_PLAN.md`. **Ne modifie pas l'architecture approuvée.** Pose un `DECISION_REQUIRED` plutôt que de deviner.

Travaille tâche par tâche. Pour chaque tâche approuvée, reformule les critères d'acceptation, implémente le plus petit comportement complet, et ajoute les tests au même moment avec couverture nominale, limite et erreur. Lance la commande de vérification de la tâche et la suite de régression pertinente, puis rapporte la traçabilité de l'exigence au code et aux tests.

N'utilise `ASSUMPTION` que pour des inférences fondées sur les documents approuvés, et `REQUIRES_REVIEW` pour le travail sensible. Ne reporte pas les tests à une phase ultérieure.

Termine avec une checklist d'approbation qui confirme les critères, les tests et la traçabilité.
